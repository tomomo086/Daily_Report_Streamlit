# 📋 警備業務日報自動化ツール

> **AI駆動開発ポートフォリオ** - Grok3、Claude4、Cursorを活用したバイブコーディング実践プロジェクト

警備業務の日報作成業務を完全自動化するWebアプリケーションです。従来手動で30分以上かかっていた日報作成を、わずか1分程度に短縮できる設計で開発しました。（※実際の運用は職場の方針により実現されませんでした）

## 🎯 プロジェクト概要

### 背景・課題
- **従来の課題**: 警備業務の日報作成が手動で時間がかかる（30分/回）
- **実際の業務実態**: 多くのスタッフが過去の日報をコピペして作成している
- **業務負荷**: 担当者選択、巡回時間記録、施設使用状況確認など複雑な作業
- **ヒューマンエラー**: 手動入力によるミス、記録漏れ
- **標準化不足**: 担当者によって記録品質にばらつき

### AI駆動開発アプローチ（バイブコーディング）
このプロジェクトは最新のAI駆動コーディング手法で構築されました：

**現在のStreamlit版（最新）:**
- **🤖 Claude Code**: 現在のStreamlit版アプリケーション開発の中核実装
- **⚡ Cursor環境**: Claude Codeとの統合開発環境でのコード生成と実行

**過去バージョン開発履歴:**
- **📱 USBポータブル版**: WinPython環境から実行していた初期バージョン
- **🧠 Claude4**: 初期要件に基づくコード実装とエラーハンドリング
- **🤖 Grok3**: アーキテクチャ設計とロジック最適化の提案
- **💻 バイブコーディング**: AI主導によるコード生成、人間は簡単な修正のみ

### 期待される業務改善効果
- ⏰ **作業時間短縮**: 30分 → 1分（96%削減の可能性）
- 🎯 **エラー削減**: 手動入力ミスの完全排除
- 📊 **標準化**: 一貫した品質の日報自動生成
- 🚀 **生産性向上**: 年間約200時間の業務時間削減見込み

### 実際の結果
- ✅ **技術的成功**: 想定通りの機能実装を達成
- ❌ **導入失敗**: 職場の理解が得られず使用禁止に
- 💡 **学習成果**: AI開発手法とバイブコーディングスキル習得

## 🎯 技術的なポイント・工夫

### リアルな巡回時間生成アルゴリズム
- **自然な時間間隔**: 15-20分の基本間隔にランダム要素を追加

- **施設利用状況連動**: 使用中の施設は巡回時間を調整してリアリティを向上
- **時間の整合性**: 開始時刻から終了まで論理的に一貫した時間生成

- **手動作業の完全代替**: コピペ＋手動修正の作業フローを自動化で置き換え
- **現実的な記録生成**: 実際の巡回記録以上の自然な内容

### 技術的チャレンジ
